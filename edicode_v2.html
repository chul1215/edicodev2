검색

1) 파일 업로드

대용량 파일은 분석 중 로딩이 있을 수 있어.
전용 EDI 집계
• 발생수(환자기준)은 “환자×EDICODE” 단위로 1건입니다(동일 환자 반복 시행은 1건).
• 금액은 반복 시행 금액을 모두 합산합니다(누적 비용).
• 1회 단가는 “해당 EDI 총액 ÷ 시행건수(라인)” 입니다(반복 시행은 시행건수로 반영).
• 미발생(미시행) 기준: 동일 진료과에서 상대 병원에 해당 EDICODE가 0건이면 전용으로 분류합니다.
• 전용 항목 칩에 진료과 코드가 표시되면, 상대 병원의 해당 진료과에서 동일 EDI가 발생했음을 의미합니다.

2) 데이터 정상성 체크

기간(min~max), 총행수/유효행수(급여구분 1,2,3,5), “제외 적용” 상태가 예상과 맞는지 확인하세요.

3) 병원별 요약 병실료/간호관리료 제외 적용

4) 진료과 비교 보드 행 클릭 → 진료과 상세

진료과 객단가 차이 (대전-유성) 대전 객단가 유성 객단가 대전 환자수 유성 환자수 EDI 구성 (공통/대전전용/유성전용)
객단가 = (병실료/간호관리료 제외 진료비) / (진료과 환자수). 상단 토글로 총액/급여/비급여 전환 가능.
